# 远程算力机加速拉取 GitHub 方案

> **场景**：协作者在中国境内的 A100 服务器上 `git clone` / `git pull`
> 我们的 GitHub 仓库速度太慢（通常 < 100 KB/s 甚至超时），影响每次更新代码的效率。
>
> 整合后只有 **一个** 仓库 `hil-serl-ImprovedVersion`（V1 / V2 / JAX baseline 全在内）。
> 本文给出 5 种加速方案，按"立即可用 → 长期最优"排序，任选一个或组合使用。

---

## 速选指南

| 你的情况 | 推荐方案 |
|---------|---------|
| 想立刻拿到代码（一次性） | **方案 A**（ghproxy 代理）|
| 想长期高速拉取（每天 git pull）| **方案 B**（Gitee 镜像）|
| 你本地拉 GitHub 很快，远程很慢 | **方案 C**（本地中转 rsync）|
| 仓库太大或一次性传输 | **方案 D**（OSS 云盘中转）|
| 服务器有梯子或 socks 代理 | **方案 E**（proxychains）|

---

## 方案 A — `ghproxy` 反向代理（最快上手，30 秒）

把 GitHub URL 换成代理 URL，**不用任何配置**，直接跑 `git clone`。

### 一次性 clone

```bash
cd ~/project
git clone https://ghproxy.com/https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git
```

也可以用：
- `https://mirror.ghproxy.com/https://github.com/...`
- `https://gh.api.99988866.xyz/https://github.com/...`
- `https://kkgithub.com/...`（直接替换 github.com）

### 已经 clone 过的仓库改 remote

```bash
cd ~/project/hil-serl-ImprovedVersion
git remote set-url origin https://ghproxy.com/https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git
git remote -v   # 验证
git pull        # 现在通过代理拉取
```

### 优缺点

| ✅ 优点 | ⚠️ 缺点 |
|---------|---------|
| 30 秒上手，无需注册 | ghproxy 偶尔抽风（503） |
| 稳定速度 5-15 MB/s | 文件 > 100 MB 可能失败 |
| 命令完全兼容 git | 公益服务，可能被关 |

**推荐用法**：作为快速救场方案，先把代码拉下来再决定要不要做更彻底的方案 B。

---

## 方案 B — 镜像到 Gitee（强烈推荐，长期最优）

**Gitee** 是国内的代码托管平台，原生速度 50+ MB/s，且支持自动同步 GitHub 仓库。

### 一次性设置（5 分钟）

#### Step 1：协作者注册 Gitee 账号
访问 [gitee.com](https://gitee.com) 注册（手机号即可）。

#### Step 2：导入 GitHub 仓库
1. 登录后右上角 `+` → `从 GitHub/GitLab 导入仓库`
2. 输入 GitHub 仓库 URL：`https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion`
3. 点击导入。Gitee 会拉取 GitHub 的全部代码到 Gitee。

#### Step 3：开启自动同步（可选但强烈推荐）
- 仓库设置 → `仓库镜像管理` → 添加镜像
- 源 URL：`https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git`
- 触发方式：定时（如每 30 分钟）

之后每当主作者 push 新版本，Gitee 自动同步，**协作者从 Gitee 拉就行**。

### 远程服务器上的使用

```bash
# 首次 clone
cd ~/project
git clone https://gitee.com/<协作者用户名>/hil-serl-ImprovedVersion.git

# 已有仓库改 remote
cd ~/project/hil-serl-ImprovedVersion
git remote set-url origin https://gitee.com/<协作者用户名>/hil-serl-ImprovedVersion.git
git pull
```

### 优缺点

| ✅ 优点 | ⚠️ 缺点 |
|---------|---------|
| **国内速度极快**（50-100 MB/s） | 需要协作者在 Gitee 注册 |
| 永久稳定，无 503 | 同步有延迟（30 分钟级） |
| 100% git 命令兼容 | 私有仓库需付费（公开免费） |
| 可挂自动同步，零维护 | — |

**推荐用法**：长期使用首选。一次设置，终身受益。

## 方案 C — 本地中转：scp / rsync 推送

如果**协作者本地的 Mac/PC 拉 GitHub 速度可以**（比如有梯子或本地网络好），可以用本地做中转，避开远程服务器直连 GitHub。

### 一次性同步（首次部署）

```bash
# 在协作者的本地机器上
cd ~/Downloads
git clone https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git

# 用 rsync 推到远程（推荐，支持断点续传 + diff 增量）
rsync -avzP --exclude='.git/objects/pack/*.pack' \
    hil-serl-ImprovedVersion/ \
    user@a100-server:~/project/hil-serl-ImprovedVersion/
```

> `-z` 启用压缩传输，`-P` 显示进度+断点续传。本地家庭宽带 100Mbps 上行约 12.5 MB/s，比远程拉 GitHub 通常快 100×。

### 增量更新（后续 `git pull` 替代）

每次主作者在 GitHub 推了新版本：

```bash
# 协作者本地：先把本地副本更新
cd ~/Downloads/hil-serl-ImprovedVersion
git pull origin main

# 然后 rsync 推到远程（仅传输 diff）
rsync -avzP --delete \
    --exclude='outputs/' --exclude='.git/objects/pack/*.pack' \
    ./ user@a100-server:~/project/hil-serl-ImprovedVersion/
```

`--delete` 让远程与本地完全一致；`--exclude='outputs/'` 防止覆盖远程的训练数据。

### 配置 SSH 别名简化命令

在协作者本地的 `~/.ssh/config` 加：

```
Host a100
    HostName <A100 服务器 IP>
    User <用户名>
    Port 22
    IdentityFile ~/.ssh/id_ed25519
    Compression yes
    ServerAliveInterval 60
```

之后命令简化为：
```bash
rsync -avzP ~/Downloads/hil-serl-ImprovedVersion/ a100:~/project/hil-serl-ImprovedVersion/
```

### 优缺点

| ✅ 优点 | ⚠️ 缺点 |
|---------|---------|
| 速度仅受本地上行带宽限制 | 需要协作者本地能拉 GitHub |
| 增量传输极快（只传 diff） | 多了一步本地操作 |
| 不依赖任何第三方服务 | — |

**推荐用法**：协作者本地有梯子时的首选。

## 方案 D — 云盘 OSS 中转（适合大文件 / 一次性传输）

打包成 tar.gz 上传到阿里云 OSS / 腾讯云 COS / 七牛云，远程服务器直接 wget。

### Step 1：本地打包

```bash
cd ~/Downloads
git clone https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git

# 打包（剔除 .git 历史可大幅减小体积）
tar --exclude='.git' -czf hilserl_repo.tar.gz hil-serl-ImprovedVersion/

ls -lh hilserl_repo.tar.gz   # 通常 600-900 MB（整合后含 4 个变体 + SurRoL_v2）
```

### Step 2：上传到云盘

**阿里云 OSS**（推荐，国内速度快）：
```bash
# 安装 ossutil
brew install ossutil   # 或 wget https://gosspublic.alicdn.com/ossutil/...

# 配置（一次性）
ossutil config

# 上传
ossutil cp hilserl_repo.tar.gz oss://<你的-bucket>/hilserl_repo.tar.gz

# 生成临时下载链接（24 小时有效）
ossutil sign oss://<你的-bucket>/hilserl_repo.tar.gz --timeout 86400
```

**腾讯云 COS / 七牛云** 同理。

**网盘替代**：百度网盘 / 阿里云盘也可以，但下载需要账号、限速、命令行工具弱。

### Step 3：远程下载并解压

```bash
cd ~/project
wget "<上一步生成的签名 URL>" -O hilserl_repo.tar.gz
tar -xzf hilserl_repo.tar.gz
ls   # 应看到 hil-serl-ImprovedVersion/
```

### 优缺点

| ✅ 优点 | ⚠️ 缺点 |
|---------|---------|
| OSS 下载速度极快（100+ MB/s） | 需要付费 OSS 账号（流量便宜） |
| 不依赖 git，可传任意文件 | 不支持增量更新（每次完整传） |
| 支持大文件（> 1 GB） | 下载完没了 .git，git 命令不能用 |

**推荐用法**：首次大批量传输。后续小更新走方案 B 或 C。

## 方案 E — 网络层代理（终极方案，需服务器有梯子）

如果远程服务器本身有梯子（如自建代理 / 公司 VPN），可以全局走代理。

### 设置 git 走代理

```bash
# 假设代理地址是 127.0.0.1:7890（v2rayN/clash 默认）

# 全局设置 git 通过 socks5 代理
git config --global http.proxy 'socks5://127.0.0.1:7890'
git config --global https.proxy 'socks5://127.0.0.1:7890'

# 验证
curl -x socks5://127.0.0.1:7890 -I https://github.com
# 期望看到 HTTP/2 200

git clone https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git
```

### 优缺点

| ✅ 优点 | ⚠️ 缺点 |
|---------|---------|
| 全局生效，所有 GitHub 操作都加速 | 需要服务器本身有梯子 |
| 速度最稳定 | 代理可能被封 |
| 长期可用 | 配置麻烦 |

**推荐用法**：你公司/学校有 VPN 时的首选。

## 最佳推荐组合

1. **协作者首次部署**：方案 A（ghproxy）快速 clone → 方案 B（Gitee）改 remote 用于长期 pull
2. **后续训练期间**：只用方案 B，`git pull` 永远秒级
3. **如果 Gitee 同步延迟太长**：临时用方案 A 直接拉最新

具体命令：

```bash
# 在 A100 服务器上：

# Step 1: ghproxy 救场首次拉取
cd ~/project
git clone https://ghproxy.com/https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git

# Step 2: 改成 Gitee 镜像（之后所有 pull 都走这里）
cd hil-serl-ImprovedVersion
git remote set-url origin https://gitee.com/<协作者用户名>/hil-serl-ImprovedVersion.git
git pull   # 验证速度
```

## 相关：仓库本身大小检查

整合后总体积约 **1.2 GB**（4 个变体 + 共享 SurRoL_v2），git 历史很浅（一次 squash commit）。如果速度不是瓶颈但磁盘吃紧，可以：

```bash
# 1. 浅克隆（只拉最新一次提交，省 90% 体积）
git clone --depth 1 https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git

# 2. 单分支
git clone --depth 1 --single-branch --branch main \
    https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git

# 3. 只拿你需要的那一格（例如只跑 V2 PyTorch）
git clone --depth 1 --filter=blob:none --sparse \
    https://github.com/LIJianxuanLeo/hil-serl-ImprovedVersion.git
cd hil-serl-ImprovedVersion
git sparse-checkout init --cone
git sparse-checkout set v2-pytorch docs results
```
