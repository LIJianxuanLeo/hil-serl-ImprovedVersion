# Touch 设备干预操作规范

适用于 HIL-SERL 训练中使用 Geomagic Touch（Phantom Omni）haptic 设备进行人工干预。

---

## 1. 按键说明

```
        ┌─────────────────┐
        │   Touch 手柄     │
        │                 │
        │  [后键 / Gray]  │  ← Button 2：切换夹爪开/关
        │                 │
        │  [前键 / White] │  ← Button 1：干预触发键（按住生效）
        │                 │
        └─────────────────┘
```

| 按键 | 功能 | 说明 |
|------|------|------|
| **Button 1（前键，白色）** | **干预触发** | **按住** = 人工接管，**松开** = 策略自主运行 |
| **Button 2（后键，灰色）** | 夹爪切换 | 每次按下切换夹爪开/关状态（toggle） |

> 训练模式（`clutch_mode=False`，当前默认）下 Button 1 是**干预触发键**，不是暂停键。  
> 录制模式（`clutch_mode=True`）下 Button 1 才是暂停/离合键，两种模式行为不同，请勿混淆。

---

## 2. 训练阶段与干预策略

训练共分三个阶段，不同阶段应采用不同的干预频率：

### 阶段一：冷启动期（0 ~ 5K steps）

**策略什么都不会，完全需要人工引导。**

- 干预频率：**80% ~ 100%**（几乎每个 episode 都介入）
- 操作目标：示范完整的 接近 → 抓取 → 抬起 动作序列
- 每次干预时长：**贯穿整个 episode**（从头到尾按住 Button 1）

### 阶段二：学习期（5K ~ 30K steps）

**策略开始尝试自主完成，但经常在抓取步骤失败。**

- 干预频率：**30% ~ 60%**（在关键时刻介入）
- 操作目标：策略自主运行，在快要失败时介入纠正
- 干预时机（见下节）

### 阶段三：精炼期（30K steps 之后）

**策略能完成大部分任务，只需偶尔纠错。**

- 干预频率：**5% ~ 20%**
- 操作目标：纠正边界情况，避免坏习惯固化

---

## 3. 干预时机（关键）

**以下情况应立即按住 Button 1 介入：**

| 情况 | 描述 |
|------|------|
| 机械臂偏离方块 > 5cm | 策略朝错误方向运动，浪费步数 |
| 机械臂在方块旁"徘徊"超过 3 秒 | 策略卡住，无法靠近 |
| 夹爪位置偏高或偏低，无法形成有效抓取 | 需要调整 Z 轴高度 |
| 方块刚被抓住但正在滑落 | 稳定当前抓取姿态 |
| episode 剩余步数不足，策略仍未接近方块 | 用剩余步数示范接近动作 |

**以下情况不建议干预（让策略自己尝试）：**

- 策略正在缓慢但正确地靠近方块
- 策略刚刚抬起方块（即使姿态不完美）
- 策略刚成功完成任务（reward ≥ 1.0）

---

## 4. 标准干预操作流程

```
① 观察策略行为（1~2 秒）
          ↓
② 判断是否需要介入（参考第3节）
          ↓
③ 按住 Button 1（手柄前键），接管控制
          ↓
④ 移动手柄引导机械臂：
     - 先对准方块 XY 位置
     - 再降低 Z 轴至方块高度
     - 按 Button 2 关闭夹爪（抓住方块）
     - 抬起手柄提升方块
          ↓
⑤ 任务完成或策略可以接管后，松开 Button 1
          ↓
⑥ 等待策略继续运行，观察下一个 episode
```

---

## 5. 夹爪操作说明

夹爪使用 **Button 2（后键）切换**，不是持续按住：

```
第一次按 Button 2 → 夹爪关闭（抓取）
第二次按 Button 2 → 夹爪打开（释放）
```

**最佳抓取姿态：**
1. 将 TCP（工具中心点）移到方块正上方约 1~2cm 处
2. 确保 XY 方向对准（误差 < 1cm）
3. 按 Button 2 关闭夹爪
4. 再向上移动手柄，将方块抬起

## 6. 终端日志判读

干预是否生效，可从终端日志确认：

```
# 策略自主运行时（无干预）：
[ACTOR] Global step 1234: Episode reward: 0.23

# 有干预的 episode（reward 通常更高）：
[ACTOR] Global step 1235: Episode reward: 1.12

# episode_metrics.csv 中应出现：
# episode_intervention = 1
# intervention_rate = 0.45  （该 episode 45% 步数为人工干预）
```

如果有干预但 `episode_intervention` 始终为 0，请检查是否按的是 Button 1（前键/白色），而非 Button 2。

## 7. 常见操作错误

| 错误 | 表现 | 解决方法 |
|------|------|---------|
| 按住 Button 1 但机械臂不动 | 手柄位移太小 | 加大手柄移动幅度；检查 `position_scale`（默认 0.2）|
| 松开 Button 1 后机械臂突然跳动 | 策略接管时动作突变 | 在策略接管前，先将手柄移回中立位置再松开 |
| 每次 episode 都是 0.03 分 | 从未成功干预，策略也没学到 | 增加干预频率，参考阶段一操作 |
| 方块每次都被碰飞 | 手柄移速过快 | 放慢操作速度，或调小 `position_scale` |
| `episode_intervention` 全为 0 | Button 1 没有生效 | 确认 `clutch_mode=False`（当前默认）；确认按的是前键 |
